* About
It's my goal to edify myself by translating into Lua 5.5 the filters and stages
which were previously translated by Zitzelsberger from lua2.5+nw into Lua 3.

I will be calling this derived work Rammel. In the future, especially if all
development goals are met, I'll send Ramsey another postcard to ask about
calling it noweb and getting the changes merged.

** Noweb 3 uses a customized version of Lua 2.5.1               :note:readme:
The introductory content of the noweb version 3 README describes what noweb
version 3 is, in brief, and why it uses Lua. It also has some cautionary content
about the futility of maintaining noweb version 3.

#+begin_quote
This is Noweb 3.  Noweb 3 is a reimplementation of Noweb using
a customized version of Lua 2.5.  Noweb 3 was conceived and
implemented in 1996 to solve two problems:

  - Noweb 2 was hard to install.
  - Without Icon, Noweb 2 can slow and hard to extend.

Advances in software and hardware have made these problems irrelevant:
Linux distributions and OSX ports have shifted installation costs from
the end user to the distribution maintainers, and those same
distributions have made it easy to build on fast Icon code instead of
slow Awk code.  Noweb 3 retains some value; it is more flexible and
easier to extend than Noweb 2, but in Fall 2015, as these words are
being written, maintaining Noweb 3 makes no sense.  In a just world,
Noweb 3 would be completely rewritten using Lua 5.  But that won't
happen.

I have used Noweb 3 in production for some years, but I continue to
call this distribution "alpha quality."  Alpha versions are numbered
by date; for example, the first version to tangle and weave itself was
alpha980616.  This alpha version is made available from github
(nrnrnr/noweb3) so that those who wish to play with Noweb 3 can do so.
#+end_quote

The reason I am interested in noweb version 3 is because (all?) filters and
pipeline stages are implemented in Lua, rather than Icon. I don't want to learn
Icon to hack on noweb version 2 filters, and it really isn't ideal to implement
advanced customizations or hacks in AWK.

* Lua 2.5.1
** Building Lua 2.5.1 requires patches
The source files "luamem.h" and "luamem.c" exist in the source repository for
Lua 2.5.1, but all other sources refer to "mem.h" and "mem.c"; the other sources
were updated by the time Lua 3 was released. A patch is to be shipped with
Rammel so Lua 2.5.1 (with the Ramsey customizations) can be built.

** Building Lua 2.5.1 requires RCS
Without RCS installed the build seems to fail.

** =diff= of Norman's lua-2.5+nw and Lua 2.5.1
The git log of Noweb 3 indicates that Norman lost track of the updates he made
to Noweb 3 at one point and simply committed everything he had ("who knows what
changes!"): a very normal occurance in software development. When I first read
that commit message I thought Norman was referring to the changes made to Lua
2.5, in total, so I created a diff. The commit message isn't about just those
changes, however; regardless, the diff lives here and is useful to have because
it can be applied as a patch to the Lua 2.5.1 sources to reproduce Lua 2.5+nw.

Five months or more since I wrote that paragraph, I realize that Dazitzel
already excised the customizations to Lua 2.5 that Ramsey made to facilitate the
original implementation of Noweb 3. Dazitzel replaced the customized version of
Lua 2.5 with the upstream Lua 2.5 and excised all the Lua 2.5 script which
required those customizations, replacing them with equivalent standard Lua.
Dazitzel also wrote Lua 3 versions of some or all Lua scripts, so Dazitzel's
existing work has saved me countless hours of learning, research, and work; Lua
3 is also closer to Lua 5, not just in version number, but if I recall there
were major changes when Lua 3 was released which will make the eventual
transition to Lua 5 easier.

** [options] Build-time Configuration Options
All the options for configuring Noweb 3 are also present in Rammel,
but implemented in Premake5 rather than Make variables.

The configuration arguments, set at the command-line, are as follows:

- LD         :: what to call to build an executable binary
- LDFLAGS    :: general flags used to build an executable
- LIBPROLOG  :: anything you need to place before listing libraries
- LIBEPILOG  :: anything you need to place after listing libraries

The =options= Org source block below defines a nested table of options; each
nested table contains the arguments to the Premake function ~newoption()~, which
takes the nested table as an argument. All the nested tables in the =opts= table
are looped over, calling ~newoption()~ for each.

**** Linker PROLOGUE and EPILOGUE variables
On Windows, the default flags for Noweb 3 were "/nologo /Fe: /link".

- =/link= :: Passes the specified option to LINK.
- =/Fe:= :: Specifies a name and a directory for the .exe file or DLL created by the compiler.
- =/nologo= :: Suppresses display of sign-on banner.

On Linux the defaults were:

- gcc :: The GNU Compiler Collection command name.
- LFLAGS :: Linker flags, which will be betwixt LIBPROLOG and LIBEPILOG.
- LIBPROLOG :: Things to prepend to the list of linker arguments.
- LIBEPILOG :: Things to append to the list of linker arguments, like =-lm=.

[[https://gcc.gnu.org/onlinedocs/gcc/Link-Options.html][=-nolibc= has the following description]]:

#+begin_quote
Do not use the C library or system libraries tightly coupled with it when
linking. Still link with the startup files, libgcc or toolchain provided
language support libraries such as libgnat, libgfortran or libstdc++ unless
options preventing their inclusion are used as well. This typically removes
=-lc= from the link command line, as well as system libraries that normally go
with it and become meaningless when absence of a C library is assumed, for
example =-lpthread= or =-lm= in some configurations. This is intended for
bare-board targets when there is indeed no C library available.
#+end_quote

versions 2.5.1 [with customizations] and 5.4.8). Third, the Rammel sources are

4. =NWMTIMEOBJS= is a list of all object files required for the =nwmtime= target. See the ~NWMTIMEOBJS~ list.
5. =FILES= is a list of files...

   bootstrapping. SRCS is a list of C code (.c and .h) which should be
   distributed; whatever .c and .h files are generated should be members of the

This subtree contains a source block that was formerly an independent Premake5

This Premake script is quite complete, if not finished. Test it a few times.

Sub-projects of Rammel must have their c or h files tangled before they can be
built, so each of these are regenerated before the project is defined. To
regenerate these files a very handsome little Lua function was written to tangle
these given a table and some optional arguments.

#+begin_src lua :noweb-ref "Rammel"
  -- tangle_c_or_h_file("example") :: notangle example.nw > example.c# tangle the default * module.
  function tangle_c_or_h_file (noweb_file_basename, root, of, mode, tangleopts)
    if not root and not of then
      of = path.join("../src/c", string.format("%s.c", noweb_file_basename))
      if not path.iscfile(of) then printf("Path: %s is erroneously not a C file!", of) end
    end
    if not of then
      of = path.join("../src/c", string.format("%s.h", noweb_file_basename))
      if not path.iscppheader(of) then printf("Path: %s is erroneously not an H file!", of) end
    end

    --[[In one call the noweb_file_basename is actually an absolute path.
    "If trailing is an absolute path, then the leading portion is ignored, and the
    absolute path is returned instead (see below for examples)."]]
    noweb_file = path.join("../src/c", noweb_file_basename)
    if not path.isabsolute(noweb_file) then
      --[[When noweb_file is contstructed from a real basename, the noweb_file
      must be given an extension.]]
      noweb_file = path.replaceextension(noweb_file, "nw")
    end

    if root then
      root = string.format("-R\"%s\"", root)
    else
      root = ""
    end
    if not mode then
      mode = ">"
    else-- I only ever use "a+" to indicate that I intend to append.
      mode = ">>"
    end

    if not tangleopts then tangleopts = "" end

    cmd = string.format([[notangle %s %s -L"#line %%L "\"%s.nw"\"%%N" %s %s %s]],
      noweb_file,
      tangleopts,
      noweb_file_basename,
      root,
      mode,
      of)
    verbosef("    %s", cmd)
    os.execute(cmd)
  end
  verbosef("Generating sources with bundled Noweb (version 2)...")
#+end_src
